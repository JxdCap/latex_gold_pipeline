# 字段规范

第一阶段统一使用中文字段名，字段顺序如下：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `编号` | 字符串 | 样本唯一编号，格式如 `样本000001` |
| `原文本` | 字符串 | A 列原始文本，空值转为空字符串 |
| `来源文件` | 字符串 | 输入 Excel 路径 |
| `来源工作表` | 字符串 | 默认读取的第一个工作表名称 |
| `来源行号` | 整数 | Excel 中的真实行号 |
| `清洗后文本` | 字符串 | 轻预洗后的文本 |
| `轻预洗标记` | 数组 | 轻预洗阶段产生的标签 |
| `是否待复核` | 布尔值 | `true/false`，不确定样本必须为 `true` |
| `分桶结果` | 字符串 | 分桶器输出的桶名 |
| `分桶依据` | 字符串 | 当前分桶命中的主要原因 |
| `分流结果` | 字符串 | 分流器输出的去向 |
| `分流依据` | 字符串 | 当前分流命中的主要原因 |

推荐枚举值：

- `轻预洗标记`：
  `空文本`、`空白规范化`、`命中疑难模式`
- `分桶结果`：
  `空白样本`、`普通文本`、`LaTeX样本`、`疑难样本`
- `分流结果`：
  `自动处理队列`、`人工复核队列`

约束：

- 导入器最少必须产出：`编号`、`原文本`、`来源文件`、`来源工作表`、`来源行号`。
- `原文本` 必须保留，不允许被覆盖。
- 导入器只做格式标准化，不做重清洗、不做分桶。
- 后续所有处理必须基于 `data/原始数据/原始样本.jsonl` 继续。
- `清洗后文本` 只做低风险整理，不做内容改写。
- `是否待复核=true` 的样本，`分桶结果` 默认应为 `疑难样本` 或 `空白样本`。
- `分流结果` 只允许由 `分桶结果` 和 `是否待复核` 决定，不允许加入复杂批次特判。
